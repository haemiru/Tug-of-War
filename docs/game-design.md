# 줄다리기 (Tug of War) - 게임 기획서

> 오징어 게임 시즌1 줄다리기 에피소드를 모티브로 한 캐주얼 웹 게임
> 기술 스택: React + Vite + Tailwind CSS

---

## 1. 게임 개요

### 1.1 컨셉

넷플릭스 드라마 "오징어 게임"의 줄다리기 에피소드를 1인용 캐주얼 웹 게임으로 재해석한다. 플레이어는 줄다리기 팀의 리더로서, 빠른 연타와 정확한 타이밍 조작을 통해 AI 상대팀을 이기고 생존해야 한다.

### 1.2 핵심 목표

- **짧고 강렬한 플레이**: 1판 60~90초 내외의 긴장감 넘치는 경험
- **간단한 조작, 깊은 전략**: 연타 + 타이밍 액션의 조합으로 누구나 쉽게 배우되 숙달이 필요한 게임성
- **오징어 게임 분위기**: 드라마의 긴장감, 시각 연출, 사운드를 최대한 반영

### 1.3 타겟 유저

| 구분 | 설명 |
|------|------|
| 1차 타겟 | 오징어 게임 팬 (10~40대), 캐주얼 웹 게임 유저 |
| 2차 타겟 | SNS 공유를 좋아하는 유저 (결과 화면 공유) |
| 플랫폼 | PC 브라우저 + 모바일 브라우저 (반응형) |
| 플레이 시간 | 1회 플레이 60~90초, 반복 플레이 유도 |

## 2. 게임 규칙

### 2.1 기본 메카닉

줄다리기는 **힘 게이지(Power Gauge)** 시스템으로 동작한다. 화면 중앙에 줄의 중심점을 나타내는 마커가 있고, 플레이어와 AI가 각각 줄을 잡아당기며 마커를 자기 쪽으로 이동시킨다.

```
[플레이어 팀] ◄──────●──────► [AI 팀]
              ← 당기기  당기기 →
```

- **줄 위치값**: -100 (플레이어 완전 승리) ~ 0 (중앙) ~ +100 (AI 완전 승리)
- **승리 임계값**: 줄 위치가 -70 이하 또는 +70 이상이 되면 해당 팀 승리
- **시간 제한**: 90초 (시간 초과 시 줄 위치로 승패 판정)

### 2.2 조작 메카닉: 3가지 액션

#### (1) 연타 (Mash) — 기본 당기기

- **PC**: `A` 키 또는 `←` 키 연타
- **모바일**: 화면 왼쪽 영역 연타 (탭)
- 연타할 때마다 줄을 플레이어 쪽으로 조금씩 당긴다
- 연타 속도에 비례하여 당기는 힘이 증가
- **피로도 시스템**: 연속 연타 시 피로도가 쌓여 힘이 점차 감소, 잠시 쉬면 회복

```
당기는 힘 = 기본 힘 × 연타 속도 보너스 × (1 - 피로도)
```

#### (2) 버티기 (Hold) — 수비

- **PC**: `Space` 키 꾹 누르기
- **모바일**: 화면 오른쪽 영역 꾹 누르기 (롱프레스)
- 누르고 있는 동안 AI의 당기기에 대한 저항력 대폭 증가
- 당기기 힘은 0이 되지만, 밀리는 속도가 크게 감소
- 피로도가 빠르게 회복됨
- **제한**: 연속 3초까지만 버티기 가능 (쿨다운 2초)

#### (3) 3걸음 전략 (3-Step) — 필살기

- **PC**: `S` 키 또는 `↓` 키
- **모바일**: 화면 아래로 스와이프
- 드라마에서 등장하는 "3걸음 작전" 재현
- **발동 조건**: 줄 위치가 +20 이상 (AI 쪽으로 밀리고 있을 때)만 사용 가능
- **효과**: 발동 후 1.5초간 줄을 크게 놓아주고(일부러 밀려남), 이후 강력한 반격으로 줄을 대폭 당김
- **위험/보상**: 발동 중 줄 위치가 +70을 넘으면 즉시 패배, 성공 시 줄 위치를 -30만큼 한 번에 당김
- **쿨다운**: 게임당 최대 2회 사용 가능

### 2.3 승리/패배 조건

| 조건 | 결과 |
|------|------|
| 줄 위치 ≤ -70 | **플레이어 승리** |
| 줄 위치 ≥ +70 | **AI 승리 (플레이어 패배)** |
| 90초 타임아웃 시 줄 위치 < 0 | **플레이어 승리** |
| 90초 타임아웃 시 줄 위치 ≥ 0 | **AI 승리** |
| 3걸음 전략 발동 중 +70 초과 | **즉시 패배** |

### 2.4 피로도 시스템

```
피로도 (0~100%)
├── 연타 시: 클릭당 +2% 증가
├── 아무 조작 안 할 때: 초당 -10% 감소
├── 버티기 중: 초당 -15% 감소 (빠른 회복)
├── 피로도 60% 이상: 당기는 힘 30% 감소 (화면 경고 표시)
└── 피로도 90% 이상: 당기는 힘 60% 감소 + 화면 붉은 틴트
```

## 3. 게임 플로우

### 3.1 전체 흐름

```
[메인 화면] → [난이도 선택] → [준비 화면] → [게임 플레이] → [결과 화면]
     ↑                                                         │
     └─────────────────── 다시하기 ─────────────────────────────┘
```

### 3.2 각 단계 상세

#### 단계 1: 메인 화면

- 게임 타이틀 "줄다리기" 표시 (오징어 게임 스타일 폰트)
- "게임 시작" 버튼
- "조작법" 버튼 (모달로 조작법 안내)
- 배경: 높은 플랫폼 위의 줄다리기 경기장 일러스트 (어두운 톤)

#### 단계 2: 난이도 선택

- 쉬움 / 보통 / 어려움 3단계 선택
- 각 난이도 옆에 간단한 설명 표시
- 선택 시 바로 준비 화면으로 이동

#### 단계 3: 준비 화면 (카운트다운)

- 드라마 연출 재현: "줄을 잡으세요" 멘트 표시 (1.5초)
- 경기장 진행 요원의 "시작!" 카운트다운: 3, 2, 1, 시작!
- 카운트다운 동안 긴장감 있는 효과음
- 간단한 조작 힌트 오버레이 표시 (첫 플레이 시)

#### 단계 4: 게임 플레이 (핵심)

- 상단: 남은 시간 표시 (90초 카운트다운)
- 중앙: 줄다리기 시각화 (줄 위치 마커, 양 팀 캐릭터)
- 하단: 피로도 게이지, 3걸음 전략 버튼 (모바일), 조작 힌트
- 게임 진행 중 실시간 줄 위치 업데이트
- 줄이 한쪽으로 크게 기울면 화면 흔들림(shake) 효과

#### 단계 5: 결과 화면

**승리 시:**
- "생존" 글자 크게 표시 (초록색, 오징어 게임 스타일)
- 클리어 시간, 3걸음 전략 사용 횟수, 최대 연타 속도 등 통계
- "다시하기" / "난이도 올리기" 버튼
- 결과 공유 버튼 (SNS)

**패배 시:**
- "탈락" 글자 표시 (빨간색)
- 패배 원인 표시 ("시간 초과", "밀려남", "3걸음 전략 실패")
- "다시하기" 버튼
- 팁 표시 ("버티기를 활용해 피로도를 관리해보세요")

## 4. 조작법

### 4.1 PC (키보드)

| 액션 | 키 | 설명 |
|------|-----|------|
| 당기기 (연타) | `A` 또는 `←` | 반복 입력할수록 강하게 당김 |
| 버티기 | `Space` (꾹) | 누르고 있는 동안 저항력 증가, 피로 회복 |
| 3걸음 전략 | `S` 또는 `↓` | 밀리고 있을 때 역전 기회 (최대 2회) |
| 일시정지 | `Esc` 또는 `P` | 게임 일시정지 |

### 4.2 모바일 (터치)

| 액션 | 조작 | 설명 |
|------|------|------|
| 당기기 (연타) | 화면 왼쪽 영역 탭 연타 | 빠르게 반복 탭 |
| 버티기 | 화면 오른쪽 영역 롱프레스 | 꾹 누르고 있기 |
| 3걸음 전략 | 전용 버튼 탭 (하단) | 조건 충족 시 활성화 |
| 일시정지 | 우측 상단 일시정지 아이콘 | 게임 일시정지 |

### 4.3 조작 영역 레이아웃 (모바일)

```

│        [⏸]          [타이머]    │  ← 상단 바
│                                 │
│                                 │
│   [줄다리기 시각화 영역]          │  ← 게임 화면
│                                 │
│                                 │
├────────────────┬────────────────┤
│                │                │
│   연타 영역     │   버티기 영역   │  ← 터치 영역
│   (탭 탭 탭)    │   (꾹 누르기)   │
│                │                │
├────────────────┴────────────────┤

```

## 5. 난이도 설계

### 5.1 AI 난이도 3단계

| 항목 | 쉬움 | 보통 | 어려움 |
|------|------|------|--------|
| AI 기본 당기기 힘 | 0.6x | 1.0x | 1.4x |
| AI 연타 속도 (CPS) | 3~5 | 5~8 | 8~12 |
| AI 버티기 사용 | 안 함 | 가끔 (30%) | 자주 (60%) |
| AI 피로도 관리 | 안 함 | 기본 | 최적 |
| AI 3걸음 대응 | 대응 안 함 | 50% 대응 | 90% 대응 |
| AI 힘 변동 패턴 | 일정함 | 약간 변동 | 파도형 공격 |
| 시간 제한 | 90초 | 90초 | 75초 |
| 3걸음 전략 횟수 | 3회 | 2회 | 1회 |

### 5.2 AI 행동 패턴

#### 쉬움 (Easy)

- AI는 일정한 속도로만 당긴다 (예측 가능)
- 버티기를 사용하지 않아 플레이어가 쉽게 밀어낼 수 있음
- 플레이어가 3걸음 전략을 써도 반격하지 않음
- **목적**: 게임 메카닉 학습용, 첫 플레이어도 승리 가능

#### 보통 (Normal)

- AI가 가끔 버티기를 사용하여 밀리지 않으려 함
- AI 당기기 힘이 간헐적으로 변동 (예측 어려움 증가)
- 플레이어의 피로도가 높을 때 공격적으로 전환
- 3걸음 전략 사용 시 50% 확률로 AI도 강하게 당김
- **목적**: 적당한 긴장감, 피로도 관리 필요

#### 어려움 (Hard)

- AI가 파도형 공격: 강하게 당기기 → 버티기 → 강하게 당기기 반복
- 플레이어의 피로도가 높을 때 집중 공격
- 3걸음 전략 사용 시 AI도 반격 (역전 성공률 하락)
- 시간 제한 짧아짐 (75초), 빠른 판단 필요
- **목적**: 숙련자 도전, 타이밍과 전략 최적화 필수

### 5.3 밸런싱 기준

- **쉬움**: 첫 플레이 승률 목표 80%
- **보통**: 3~5회 플레이 후 승률 목표 50%
- **어려움**: 10회 이상 플레이 후 승률 목표 30%
- 각 난이도에서 "3걸음 전략"을 적절히 사용하면 승률 +20% 상승 목표

```
┌─────────────────────────────────────────┐
│  [⏸]          남은 시간: 01:15          │  ← 상단 바
├─────────────────────────────────────────┤
│                                         │
│    난이도: 보통          3걸음: 2/2      │  ← 정보 바

│  플레이어 팀        ●        AI 팀      │  ← 줄 위치 인디케이터
│  ├──────────────────┼──────────────┤    │
│  ██████████████████░░░░░░░░░░░░░░░░    │  ← 줄 게이지 바
│                                         │
│  🧑🧑🧑 ◄════════════════► 🧑🧑🧑    │  ← 캐릭터 시각화

│                                         │

```

**구성 요소 상세:**

| 요소 | 설명 |
|------|------|
| 상단 바 | 일시정지 버튼, 남은 시간 (큰 숫자) |
| 정보 바 | 현재 난이도, 3걸음 전략 남은 횟수 |
| 줄 위치 인디케이터 | 중앙 마커(●)가 좌우로 이동, 위험 구간은 빨간색 |
| 줄 게이지 바 | 플레이어(파랑) vs AI(빨강) 색상, 중앙 기준으로 밀고 당김 시각화 |
| 캐릭터 시각화 | 양 팀 캐릭터(초록 체육복)가 줄을 잡고 있는 모습, 줄 위치에 따라 좌우 이동 |
| 절벽/플랫폼 | 양 팀 사이에 절벽 연출 (패배 팀이 떨어지는 곳) |
| 피로도 게이지 | 0~100% 바, 60% 이상 노란색, 90% 이상 빨간색 |
| 조작 영역 | 모바일에서 터치 영역 구분, PC에서는 키보드 힌트 표시 |

## 7. 오징어 게임 연출 요소

### 7.1 시각 연출

| 연출 | 설명 | 구현 방법 |
|------|------|-----------|
| **핑크 가드 진행 요원** | 게임 시작 시 "줄을 잡으세요", "시작!" 멘트 표시 | 화면 상단에 말풍선 오버레이 |
| **초록색 체육복** | 양 팀 캐릭터 모두 초록 체육복 + 번호 착용 | CSS 기반 캐릭터 일러스트 또는 SVG |
| **절벽 연출** | 경기장 양쪽에 높은 절벽, 아래는 어둠 | 배경 일러스트 + 패럴랙스 효과 |
| **패배 시 추락 애니메이션** | 진 팀이 절벽 아래로 떨어지는 연출 | CSS 애니메이션 (translateY + opacity) |
| **줄 위치 위험 경고** | 줄이 +50 이상 밀릴 때 화면 가장자리 빨간 비네팅 | CSS box-shadow inset 효과 |
| **화면 흔들림** | 줄이 급격히 움직일 때 화면 미세 흔들림 | CSS transform + animation |
| **조명 효과** | 경기장 위 조명 (핑크/화이트 톤) | CSS gradient + 깜빡임 효과 |

### 7.2 3걸음 전략 연출

3걸음 전략은 드라마의 가장 극적인 순간을 재현하는 핵심 연출이다.

**발동 시퀀스:**

1. **발동 순간** (0.0초)
   - 화면 슬로우 모션 효과 (게임 속도 50% 감속)
   - "3걸음!" 텍스트 크게 표시 (화면 중앙)
   - 짧은 진동 피드백 (모바일)

2. **줄 놓아주기** (0.0~1.0초)
   - 플레이어 팀이 일부러 뒤로 3걸음 물러남 (캐릭터 애니메이션)
   - 줄 위치가 AI 쪽으로 빠르게 이동 (+20 추가)
   - 화면 긴장감 증가: 빨간 비네팅 강화

3. **역전 당기기** (1.0~1.5초)
   - "당겨!" 텍스트 표시
   - 플레이어 팀이 강하게 당기며 줄 위치 대폭 이동 (-50)
   - 성공 시: 화면 플래시 + 환호 효과
   - 실패 시: 줄 위치 +70 초과 → 즉시 패배 → 추락 애니메이션

### 7.3 사운드 & 음향 효과

| 장면 | 효과음 |
|------|--------|
| 메인 화면 | 긴장감 있는 배경 음악 (낮은 드론 사운드) |
| 카운트다운 | "삐-삐-삐-삐이이" 경기 시작 신호음 |
| 연타 | 줄 팽팽해지는 마찰음 (클릭당) |
| 버티기 | 끼익- 발이 미끄러지는 소리 |
| 3걸음 전략 | 드라마틱한 스팅어 음악 + 발걸음 소리 |
| 밀릴 때 | 신발이 바닥에 끌리는 소리, 긴장 음악 증가 |
| 승리 | 환호성 + 승리 팡파레 |
| 패배 | 비명 + 추락음 + 정적 |
| 위험 구간 진입 | 심장 박동 소리 (줄 위치 ±50 이상) |

> 참고: 초기 버전에서는 Web Audio API 또는 무료 효과음 라이브러리 활용. 사운드는 설정에서 ON/OFF 가능하게 구현.

### 7.4 카운트다운 연출

게임 시작 전 카운트다운은 드라마의 분위기를 살리는 핵심 요소이다.

```
[0.0초] 화면 어두워짐 + "줄을 잡으세요" 텍스트 (1.5초 유지)
[1.5초] "줄을 잡으세요" 사라짐
[2.0초] "3" 크게 표시 + 효과음
[3.0초] "2" 크게 표시 + 효과음
[4.0초] "1" 크게 표시 + 효과음 (피치 올라감)
[5.0초] "시작!" + 경기 시작 사이렌 + 게임 시작
```

### 7.5 핵심 분위기 요소

- **색상 팔레트**
  - 배경: 어두운 회색/검정 (#1a1a2e, #16213e)
  - 메인 핑크: #ED1B76 (오징어 게임 시그니처 핑크)
  - 초록 체육복: #00B894
  - 승리 초록: #00FF41
  - 패배/위험 빨강: #FF0000
  - 줄 색상: #D4A574 (밧줄 갈색)

- **폰트**
  - 타이틀: 둥근 고딕체 (Jua, Black Han Sans 등)
  - UI 텍스트: Pretendard 또는 Noto Sans KR
  - 숫자/카운트다운: 고정폭 Bold (D2Coding, monospace)

- **오징어 게임 상징 요소**
  - 원(○), 삼각(△), 사각(□) 도형을 UI 장식 요소로 활용
  - 핑크 가드 마스크 아이콘을 일시정지 버튼 등에 활용
  - 명함 스타일 디자인 (앞면 ○△□, 뒷면 전화번호)을 메인 화면에 차용

## 부록: 기술 구현 참고사항

### A. 상태 관리 구조

```
GameState {
  phase: 'menu' | 'difficulty' | 'countdown' | 'playing' | 'result'
  difficulty: 'easy' | 'normal' | 'hard'
  ropePosition: number        // -100 ~ +100 (0이 중앙)
  playerFatigue: number       // 0 ~ 100
  timeRemaining: number       // 초 단위
  threeStepCount: number      // 남은 3걸음 횟수
  threeStepActive: boolean    // 3걸음 발동 중 여부
  isHolding: boolean          // 버티기 중 여부
  holdCooldown: number        // 버티기 쿨다운 남은 시간
  result: 'win' | 'lose' | null
  stats: {
    clearTime: number
    maxCPS: number
    threeStepUsed: number
    fatigueManagement: 'S' | 'A' | 'B' | 'C'
  }
}
```

### B. 게임 루프 (60fps)

```
매 프레임:
1. 플레이어 입력 처리 (연타, 버티기, 3걸음)
2. 플레이어 힘 계산 (입력 × 피로도 보정)
3. AI 힘 계산 (난이도별 패턴)
4. 줄 위치 업데이트 (ropePosition += playerForce - aiForce)
5. 피로도 업데이트
6. 승패 판정
7. UI 렌더링
```

### C. CPS (Clicks Per Second) 계산

```
최근 1초간의 클릭 수를 추적하여 CPS 계산
→ CPS가 높을수록 당기는 힘 증가 (상한선: 15 CPS)
→ 결과 화면에서 최대 CPS 표시
```
